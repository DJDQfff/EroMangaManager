# 结构调整

* Reader页面翻页，一开始是设计的绑定到压缩文件的各个Entry，这样可以节省资源，只在发生切换页面的时候加载图像，但这样会发生两个问题：
  
  1. 解码很慢
  
  2. 有一个bug，压缩文件的ZipArchive类（标准dotnet库或者别的库）不是线程安全，在翻页时候加载图像，就需要多线程解压缩图像，因此线程报错。
  
       现在改成了提前解压文件获取图像，每个页面直接绑定到图像，不需要SelectionChange事件
